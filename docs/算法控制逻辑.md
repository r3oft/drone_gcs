# 算法控制任务整体路线

## 1. 系统总体Pipeline

由于需要将YOLO计算和无人机隔离开，鉴于局域网无线通信固有的网络抖动特性，以及视觉闭环控制对系统延迟的严苛容忍度，PC 地面站软件层面需要采用**异步多线程非阻塞模型（Asynchronous Non-blocking Model）**。

整个地面站控制 Pipeline 可分为五个任务模块。各模块之间通过明确定义的内存接口传递数据：

1. **[物理感知层]** $\xrightarrow{\text{HTTP/TCP}}$ **模块一（异步流媒体采集）**：提取绝对物理时间戳最新的图像矩阵。
2. **模块一** $\xrightarrow{\text{RGB Tensor}}$ **模块二（OBB 位姿解算）**：执行 YOLO 推理，输出目标像素坐标 $(u,v)$ 与归一化偏转角 $\theta$。
3. **模块二** $\xrightarrow{(u,v,\theta)}$ **模块三（视觉伺服控制）**：将二维图像误差非线性映射至三维机体坐标系，解算期望线速度与角速度。
4. **模块五（全局状态机）**：作为主控中枢，统筹各模块时序，评判状态跃迁，并实施异常状态的容错拦截。
5. **模块五** $\xrightarrow{\text{核准指令}}$ **模块四（底层协议映射）**：封装 MAVLink 协议字节流，经 UDP 链路透传至底层 Pixhawk 执行机构。

---

## 2. 各模块任务说明和程序接口

### 任务一：低延迟视觉流异步采集模块

**1. 任务定位**

传统同步视频流读取机制在底层依赖先进先出（FIFO）缓冲池。在网络吞吐量波动时，极易对历史帧产生积压，导致控制系统引入相位延迟。本模块旨在通过多线程技术强制清空底层缓冲，确保下游视觉感知算法获取的始终是端到端延迟最小的当前物理帧。

**2. 核心算法逻辑**

* 在类初始化时派生独立的后台守护线程（Daemon Thread）。该线程包含无限循环，高频调用底层 C++ 接口 `cv2.VideoCapture.grab()`。此操作仅从网卡剥离数据指针而不执行高耗时的 JPEG 解码，从而极速消耗网络积压。
* 仅当主干程序发起图像请求时，获取线程互斥锁（Mutex Lock），调用 `retrieve()` 对当前指针指向的最新一帧进行解码。此机制可有效规避多线程内存竞争导致的图像撕裂（Tearing）。

**3. 模块接口 (`class ZeroLatencyStreamer`)**

* **核心依赖**：`cv2`, `threading`
* `__init__(self, stream_url: str)`：构造函数。初始化流媒体网络套接字，并启动后台 I/O 守护线程。
* `get_latest_frame(self) -> numpy.ndarray | None`：数据输出接口。返回解码后的 RGB 图像张量（维度为 $[H, W, 3]$）。若流媒体断连或未就绪，则严格返回 `None`。

---

### 任务二：旋转目标检测与位姿特征提取模块

**1. 任务定位**

利用 PC 端的 GPU，实时对输入图像进行推理，提取特定目标（目标纸盒或对应颜色投放区）的高精度定向边界框（Oriented Bounding Box, OBB）及二维几何位姿参数。

**2. 核心算法逻辑**

* **网络预热与硬件加速**：加载微调后的 YOLO-OBB 权重，强制开启半精度浮点（FP16）与 GPU 后端加速。系统初始化时需输入全零张量执行前向传播预热（Warm-up），消除实飞时首帧显存分配引发的控制时延突刺。
* **对称几何学角度归一化**：叉车式夹爪与矩形纸盒具有典型的 $C_2$ 旋转对称性（即旋转 $\pi$ 弧度物理等效）。网络输出的绝对偏转角必须经过模运算（Modulo Arithmetic）折叠处理，强制映射至距当前机体航向最近的相对路径区间（如 $[-\frac{\pi}{4}, \frac{\pi}{4}]$ 或 $[-\frac{\pi}{2}, \frac{\pi}{2}]$，视具体对齐策略而定），从而避免系统生成大于 $90^\circ$ 的冗余偏航机动。

**3. 模块接口规范 (`class TargetPoseEstimator`)**

* **核心依赖**：`ultralytics`, `torch`, `numpy`
* `__init__(self, weights_path: str, conf_threshold: float = 0.6)`：加载计算图引擎并设定非极大值抑制（NMS）的置信度下限阈值。
* `process_frame(self, frame: numpy.ndarray, target_cls_id: int) -> dict | None`：推理接口。输入当前帧与目标类别标签。若检测成功，返回特征字典 `{"u": float, "v": float, "theta": float, "conf": float}`；若未达置信度阈值，返回 `None`。

### 任务三：运动学坐标映射与视觉伺服闭环控制模块

**1. 任务定位**

构建基于图像的视觉伺服（Image-Based Visual Servoing, IBVS）律。将图像平面的离散像素/角度误差，非线性映射为无人机在局部机体坐标系（Body Frame）下的平滑期望速度向量，并施加飞行安全包线限制。

**2. 核心算法逻辑**

* **机体系误差构建**：基于下视（BEV）正交投影假设，设图像光学中心为 $(u_0, v_0)$。纵向像素误差 $E_x = v_0 - v$ 映射至前向速度；横向像素误差 $E_y = u - u_0$ 映射至侧向速度；偏航误差 $E_{yaw} = 0 - \theta$ 映射至角速度。
* **去积分化比例-微分控制（No-I PD Control）**：多旋翼飞行器的动力学模型在速度至位置的传递函数中自带积分环节。若在外部控制律中引入积分控制项（I），将不可避免地导致积分饱和（Integral Windup）与低频发散振荡。因此，各解耦轴仅构建 PD 控制器计算补偿量。
* **非线性死区与绝对饱和限幅**：构建静态死区（Deadband），当目标极化误差极小时输出零向量，抑制稳态高频激振。最终计算的期望速度需经过硬饱和限幅（如 $V_{max} = \pm 0.3 \text{ m/s}$），以防过大控制量引发机架剧烈俯仰/横滚，破坏光学垂直投影假设。

**3. 模块接口规范 (`class VisualServoController`)**

* **核心依赖**：`math`, `time`
* `__init__(self, kp: list, kd: list, deadband: list, max_vel: list)`：初始化三轴独立的控制增益矩阵、死区参数及速度限幅边界。
* `compute_velocity(self, target_pose: dict, center_u: float, center_v: float, dt: float) -> tuple[float, float, float]`：解算接口。传入特征状态与微积分时间步长，返回限幅后的机体系期望速度元组 `(V_x, V_y, \omega_z)`。

### 任务四：MAVLink 底层协议抽象与执行器通信模块

**1. 任务定位**

屏蔽底层二进制异步串行协议的复杂性，在地面站与 Pixhawk 飞控间建立高可用性的 UDP 遥测链路，提供结构化的闭环速度操纵与末端执行机构（夹爪）的硬件状态覆写接口。

**2. 核心算法逻辑**

* **相对速度指令流注入**：构造 `#84 SET_POSITION_TARGET_LOCAL_NED` 报文。协议坐标参考系必须显式声明为 `MAV_FRAME_BODY_NED`，该坐标系严格独立于地球磁场，确保给定速度恒等价于“沿机体当前航向矢量运动”。利用掩码（`type_mask=0x07C7`）剥夺底层的绝对位置计算权，实行纯速度内环接管。
* **硬件通道覆写（Channel Override）**：构造 `#76 MAV_CMD_DO_SET_SERVO` 报文，向飞控指定的 AUX 端口（如通道 9）注入具有特定占空比的 PWM 脉宽信号，以驱动降压 PDB 供电的双舵机夹爪。*(注：需在飞控 Mission Planner 中将该通道的 `SERVOx_FUNCTION` 强制置为 0，禁用原生混控器干扰)*。

**3. 模块接口规范 (`class MavlinkCommander`)**

* **核心依赖**：`pymavlink` (`mavutil`)
* `__init__(self, connection_string: str)`：建立 UDP 套接字（如 `"udp:192.168.4.1:14550"`），执行阻塞挂起直至捕获首个飞控心跳包（Heartbeat）。
* `send_body_velocity(self, vx: float, vy: float, vz: float, yaw_rate: float) -> None`：封包下发前、右、下三轴线速度及偏航角速度。
* `set_gripper_state(self, is_open: bool) -> None`：解析布尔状态并映射为物理 PWM 微秒宽度下发至指定继电器通道。

### 任务五：全局有限状态机与容错调度中枢

**1. 任务定位**

作为系统的顶层主调循环，统筹调度上述四个模块的内存流转，依据时序反馈评判飞行阶段的有限状态机（FSM）跃迁，并根据设计的Corner Case设置系统物理失效保护（Failsafe）。

**2. 核心算法逻辑**

* **时序防抖跃迁准则（Debouncing Logic）**：状态跃迁（如由“平面伺服对齐”切入“垂直盲降抓取”）的充要条件严禁基于单帧判定。算法需维护积分时间窗，当且仅当任务三输出的修正速度因满足死区条件而**连续 $T$ 秒（如 1.5 秒）恒等于零**时，方可判定机体动能已完全消散，准许触发降落动作。
* **目标失联看门狗（Target Loss Watchdog）**：高频监测感知模块的返回值。
* **一级干预（短期遮挡或抖动，$< 1.0\text{s}$）**：截断伺服控制器输出，向底层下发全零速度矢量，促使机体实施原地制动悬停。
* **二级干预（完全丢失视野，$> 3.0\text{s}$）**：阻断当前控制律闭环。强制回退状态机至全局搜索模式（CRUISE_SEARCH），并下发恒定的负向 $V_z$ 速度（即控制机体垂直爬升），旨在通过增加相对高度来扩大光学系统视场角（FOV），重启目标捕获序列。
* **后期拓展**： 开发后期可根据调试时遇到的corner case，进行状态机的补充和完善。


* **全局循环频率钳制（Tick Rate Limiting）**：在主控制流循环边界处实施休眠挂起策略，将整个流水线的迭代时钟刚性钳制于 $10\text{Hz} \sim 15\text{Hz}$ 区间。此举可规避算力溢出导致的数据报文洪泛，防止底层 ESP-01S 串行网桥发生缓冲区溢出崩溃。

## 3. 全链路Pipeline伪代码

这里用python给出主程序的伪代码，展示了任务一至任务四是如何进行级联的：

```python
import time
from enum import Enum

# --- 1. 模块依赖注入与例化 ---
vision_io = ZeroLatencyStreamer("http://192.168.4.2:81/stream")
perception = TargetPoseEstimator("weights/yolov8n-obb-custom.pt", conf_threshold=0.6)
controller = VisualServoController(kp=[0.003, 0.003, 0.6], kd=[0.001, 0.001, 0.1], 
                                   deadband=[15, 15, 0.05], max_vel=[0.3, 0.3, 0.5])
mavlink = MavlinkCommander("udp:192.168.4.1:14550")

# --- 2. 状态机枚举与系统寄存器初始化 ---
class FlightState(Enum):
    CRUISE_SEARCH = 1
    ALIGN_TARGET = 2
    DESCEND_GRAB = 3
    # ...后续状态省略

current_state = FlightState.CRUISE_SEARCH
target_class_id = 0  # 初始任务：搜寻目标物体
last_seen_time = time.time()
align_stable_start = 0
last_control_time = time.time()

# --- 3. 系统总线主循环 ---
while True:
    current_time = time.time()
    dt = current_time - last_control_time
    last_control_time = current_time

    # [执行任务一] 获取物理世界时域截面
    frame = vision_io.get_latest_frame()
    if frame is None:
        continue

    # [执行任务二] 张量前向解算与几何图元提取
    target = perception.process_frame(frame, target_class_id)

    # [执行任务五 - 异常子程序] 看门狗失效保护拦截
    if target is None:
        time_elapsed = current_time - last_seen_time
        if time_elapsed > 3.0:
            current_state = FlightState.CRUISE_SEARCH
            mavlink.send_body_velocity(0.0, 0.0, -0.2, 0.0) # Vz=-0.2 m/s 爬升扩大FOV
        elif time_elapsed > 0.5:
            mavlink.send_body_velocity(0.0, 0.0, 0.0, 0.0)  # 紧急刹车悬停
        time.sleep(0.06)
        continue
    
    last_seen_time = current_time # 目标就绪，重置看门狗计时器

    # [执行任务三] 运动学伺服偏导计算
    vx, vy, vyaw = controller.compute_velocity(target, center_u=320, center_v=240, dt=dt)

    # [执行任务五 - 核心子程序] 状态机跃迁路由
    if current_state == FlightState.CRUISE_SEARCH:
        mavlink.send_body_velocity(0.1, 0.0, 0.0, 0.2) # 微速前飞并执行自旋扫描
        current_state = FlightState.ALIGN_TARGET
            
    elif current_state == FlightState.ALIGN_TARGET:
        mavlink.send_body_velocity(vx, vy, 0.0, vyaw)
        
        # 判定动能收敛与防抖跃迁
        if vx == 0.0 and vy == 0.0 and vyaw == 0.0:
            if align_stable_start == 0: 
                align_stable_start = current_time
            elif current_time - align_stable_start > 1.5:
                current_state = FlightState.DESCEND_GRAB
        else:
            align_stable_start = 0

    elif current_state == FlightState.DESCEND_GRAB:
        mavlink.send_body_velocity(vx * 0.5, vy * 0.5, 0.15, 0.0) # 维持平面纠偏的同时注入下降速度
        # [逻辑预留]: 接入高度计/激光测距反馈判定触地，随后触发 mavlink.set_gripper_state()

    # [执行任务五 - 流控制节流]
    time.sleep(0.06) # 约维持 15Hz 控制循环，避免通信拥塞

```
