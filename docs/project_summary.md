# 《基于视觉识别的微型物流无人机系统》项目逻辑总结

## 一、 项目定位与核心运转逻辑
项目并没有盲目追求在室外复杂非结构化环境中的全自主探索，而是立足于微型无人机在算力、载荷和供电上的物理约束，进行主动的“降维”设计。
*   **应用场景界定**：将场景严格限制在**“规则明确、边界清晰的固定室内场地”**，把核心精力聚焦于跑通“飞行-感知-决策-控制-执行-反馈”的多模块协同闭环。
*   **全局流转逻辑**：依托有限状态机（FSM），系统按照“进场自检 $\rightarrow$ 粗飞转移至货物区 $\rightarrow$ 局部视觉精对准 $\rightarrow$ 盲降收取 $\rightarrow$ 闭环结果确认 $\rightarrow$ 粗飞转移 $\rightarrow$ 投递 $\rightarrow$ 自主离场”的串行工作流推进。
*   **导航降维逻辑**：摒弃了昂贵且部署繁琐的室内高精度定位系统（如UWB或动捕），采用**“预定义区域定距粗飞转移 + 局部区域视觉精细对准”**的平替策略，用视觉弥补导航精度，大幅降低了硬件成本与基建依赖。

## 二、 系统架构设计（分层解耦逻辑）
为了解决机载算力薄弱、硬件模块间易产生强耦合干扰的痛点，系统采用了**“端云协同”**与**“软硬解耦”**的分层物理架构：
1.  **决策层（云端大脑）**：部署于**地面站 PC**。承担高算力负荷，负责接收 WiFi 图传、运行重负载的深度学习视觉算法（YOLO）、计算视觉伺服控制量，并统筹全局任务状态机。
2.  **飞行控制层（运动底座）**：核心为 **Pixhawk 2.4.8**。不参与高层业务逻辑计算，仅专注于底层的姿态维稳、定点悬停，并严格被动执行地面站下发的“机体系速度指令”。
3.  **任务执行层（末端触觉）**：核心为 **树莓派 Pico 2**（独立 MCU）。接管双舵机驱动与红外传感器状态读取。**解耦优势**：剥离末端机构控制，避免了高频动作指令或引脚中断占用飞控系统的宝贵 I/O 资源或干扰飞行主线程。
4.  **通信层（多链路冗余）**：物理隔离为三条链路。WiFi 专传图像流，数传专传 MAVLink 控制指令与状态，RC 遥控链路保留最高权限的人工保底接管。
5.  **供电层（物理隔离）**：将高低功率、动力与动作逻辑独立分开供电，防止大电流设备（如起飞电机突变或舵机堵转）导致飞控电压瞬间跌落而死机。

## 三、 关键技术方案选型（权衡逻辑）
项目在关键技术点的选型上，紧扣“物流动作的刚性需求”与“安全性底线”：

### 1. 视觉算法选型：为何选“YOLOv8-OBB（旋转目标检测）”？
*   *淘汰方案*：传统CV极易受旋翼高频阴影干扰；AprilTag缺乏对无标签自然物体的泛化能力；标准YOLO输出水平边界框（HBB），当目标纸盒倾斜时，为了包裹目标边界框会膨胀，导致**“几何中心发生漂移”**。
*   *入选逻辑*：由于无人机底盘采用“叉车式抓取”，机构必须获知精确的**偏航角（Yaw）**以进行航向切入。OBB 算法能输出带有倾斜角的五维张量，多边形完美贴合纸盒物理边缘，彻底消除中心漂移，满足了末端机构对“各向异性进叉”的严苛空间约束。其庞大的算力开销则通过卸载至地面站完美规避。

### 2. 末端机构选型：为何选“叉式托举机构”？
*   *淘汰方案*：电磁铁断电货物直接砸落（极其危险）；Push-Push机构对视觉对位精度要求过于苛刻；机械手重量大、机械结构复杂，抓取时容易破坏微型机重心的平衡。
*   *入选逻辑*：叉式托举机构对位容差适中，且最重要的是具备**天然的“掉电保护”失效安全特性**。在突发断电或飞行颠簸时，配合机架的物理限位，货物也不会轻易脱落。

## 四、 软件控制与任务流转逻辑
软件系统的内部运转注重“防迟滞”与“防抖动”，绝不盲目信任单次测量结果：
1.  **抗延迟视觉流逻辑**：针对 WiFi 图传不可避免的网络抖动和缓存积压，系统采用**“异步多线程非阻塞”**拉流机制。主控制线程持续清空底层缓冲区，只提取“最新一帧”进行解码，最大程度压榨端到端延迟，防止视觉控制滞后。
2.  **视觉伺服控制逻辑**：控制律采用**无积分项（I）的离散 PD 控制**。将图像的横、纵、偏航误差映射为**机体系**下的平移与角速度。刻意剔除积分项并加入死区与限幅，是为了防止由通信延迟引发的积分饱和与近距离高频震荡。
3.  **状态跃迁防抖逻辑**：系统的关键高危动作（如盲降、抓取、释放）不轻信“单帧识别结果”。从“对齐”状态切入下一步前，系统要求目标必须在规定时间内**持续、稳定地满足误差容忍阈值**，方可触发下一阶段。
4.  **硬件动作闭环逻辑**：抓取/投递指令下发给 MCU 后，必须通过底层的两路**红外传感器**进行物理层面的二次校验，向状态机明确回传成功/失败的 Flag，形成完整的控制闭环。

## 五、 异常兜底与安全容错逻辑 (Fail-Safe)
作为室内高危测试平台，系统构建了严密的异常保护网：
1.  **视觉感知容错**：
    *   *短时丢失目标*：控制律立即输出零速度指令，无人机**原地悬停**等待。
    *   *长时丢失目标*：触发状态机强制干预，退回搜索状态，指挥无人机执行**爬升**动作，拉高相对高度以扩大相机视场重新搜索。
2.  **动作执行容错**：若红外校验显示动作（抓取/投递）失败，系统不会宕机锁死，而是根据设定的阈值触发有限次的**自动重试机制**。
3.  **多链路失效降级机制（参照系统表2）**：
    *   系统确立了“RC遥控具备最高人工接管权限”的安全原则。
    *   系统实时轮询 RC 遥控、数传、WiFi 的存活状态组合。若图传/数传异常，系统将悬停或听从 RC 遥控接管。
    *   **终极兜底策略**：一旦检测到“数传控制指令”与“RC遥控”**双双断连**（即系统彻底失控失联），无人机将强制触发最高保底防线——**“自控悬停5秒后执行 RLD（返航/降落/停桨）”**，严防飞丢或撞毁。
