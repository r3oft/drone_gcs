# 物流无人机地面站控制软件 —— 开发进度汇报

> 报告截止：2026 年 4 月 9 日
> 项目名称：基于视觉识别的微型物流无人机系统设计与实现
> 模块归属：PC 地面站控制软件（drone_gcs）

---

## 一、 项目总体概况

本项目为物流无人机系统中的地面站控制软件（drone_gcs），承担深度学习视觉推理、PD 视觉伺服闭环控制以及全局状态机调度等核心计算任务。该项目采用 Python 语言开发，以五大核心模块级联的 Pipeline 架构实现从图像采集到飞行控制指令下发的完整闭环。

当前软件系统已完成 **Phase 0（基础框架搭建）** 和 **Phase 1（核心算法离线开发）** 两个阶段的全部开发工作，并已进入数据集采集与标注的初期阶段。

项目的所有代码均在<https://github.com/r3oft/drone_gcs>上进行了开源，目前更新主要在dev branch上进行，待各功能模块开发完善后，会作为 v1.0 合并到main branch上。

### 2.3 Phase 1 —— 核心算法离线开发（已完成）

Phase 1 阶段聚焦于视觉伺服控制闭环中硬件依赖度最低的核心算法模块，实现了从目标位姿解算到速度指令输出的完整数据通路。完成的模块包括：

#### （1）几何运算工具库（`utils/geometry.py`）

提供四个纯函数，服务于视觉伺服 Pipeline 的坐标变换与信号处理：

- **`normalize_obb_angle()`**：基于 C₂ 旋转对称性，将 YOLO-OBB 输出的偏转角折叠至最小等效区间 [-π/4, π/4]，避免无人机执行冗余的大角度偏航机动
- **`pixel_to_body_error()`**：基于下视正交投影（BEV）假设，将像素平面误差映射为机体坐标系前向/侧向误差
- **`apply_deadband()`**：静态死区滤波器，抑制稳态高频激振
- **`clamp()`**：硬饱和限幅函数，约束输出在安全包线内

测试用例：46 个。

#### （2）PD 视觉伺服控制器（`core/servo_controller.py`，M3 模块）

实现了三轴解耦的无积分项比例-微分（PD）控制器，将图像平面的像素/角度误差映射为机体系期望速度向量。关键设计特征包括：

- **去积分化控制律**：严格采用 PD 控制，规避多旋翼速度-位置传递函数中自带积分环节与外部 I 项叠加导致的积分饱和与低频振荡问题
- **误差空间死区**：在 PD 计算之前施加死区滤波，确保误差收敛时 P 项和 D 项同时归零，为 FSM 防抖跃迁提供确定性判据
- **dt 异常防护**：当帧间时间步长 dt ≤ 0 时自动跳过微分项计算，防止除零或无穷大导数
- **状态重置接口**：提供 `reset()` 方法，在 FSM 状态切换时清空微分项历史，避免跨阶段的控制量突变

测试用例：32 个。

#### （3）YOLO-OBB 推理管线（`core/perception.py`，M2 模块）

封装了基于 ultralytics YOLOv8-OBB 的旋转目标检测与位姿特征提取管线，支持双类别检测（取货区 pickup_zone / 投递区 delivery_zone）。关键设计特征包括：

- **GPU 预热机制**：模型加载后立即执行全零张量前向传播，消除运行时首帧的 CUDA 显存分配延迟突刺（约 200-500ms）
- **类别动态筛选**：通过 `target_cls_id` 参数实现目标类别的运行时切换，M2 模块本身保持无状态设计，类别路由逻辑完全由上游 FSM 驱动
- **C₂ 角度归一化**：推理结果经过对称性折叠后输出，确保与下游 PD 控制器的坐标约定一致
- **Mock 测试策略**：单元测试中通过 `unittest.mock` 构造合成检测结果张量，实现脱离 GPU 和真实权重文件的完整管线验证

测试用例：18 个。

#### （4）可视化调试工具（`utils/visualization.py`）

提供 Headless 模式的调试 HUD 叠绘工具，支持 OBB 旋转框绘制、误差向量箭头、半透明状态面板叠加、视频录制和帧快照保存。采用 Headless 架构的原因在于：项目环境中标注工具 X-AnyLabeling 携带的 PyQt5 与标准 `opencv-python` 内嵌的 Qt 库存在 xcb 插件冲突，因此全程使用 `opencv-python-headless`，以视频录制和 PNG 快照替代实时 GUI 显示。

测试用例：27 个。

## 三、 方案迭代说明

开发过程中，系统设计方案经历了三次迭代优化：

| 版本 | 核心变更 | 动因 |
|:---|:---|:---|
| v0.1 | 初始方案：单摄像头 + 单类别（货箱） | 初始设计 |
| v0.2 | 双摄像头 + 双类别（货箱 + 投递区） | 发现抓取后货箱遮挡中心摄像头视野 |
| **v0.3（当前）** | **单摄像头 + 双类别（取货区 + 投递区）** | 自主抓取货箱难度过大，降级为取货区人工装填方案 |

v0.3 版本的主要改动包括：以场地内蓝色胶带标记的"取货区"替代自主抓取货箱，无人机视觉对准取货区后降落，由人工完成货箱装填，红外传感器确认装填完成后起飞转运。此方案使得单摄像头（外侧安装，视野不受货箱遮挡）即可覆盖全部视觉检测需求，降低了系统复杂度。
